Introduction¶

This dataset originally came from the kaggle challenge. The purpose of this challenge was to predict which movie will be successful before it is released. So dataset includes various features related to movie industry. We are not going to make predicting model here but checking which feature contribute for success will be an interesting and valuable question. Mainly, we are going to focus on the dataset itself and try to find out some insights from that dataset. The process will be start from checking the dataset itself to data wrangling, exploring, and visualizing. Pandas will be used for data wrangling and matplotlib will be used for viusalizing. Finally, we will show our findings and wrap it up.

Properties

I wanted to check the detail information about each properties but cannot find detailed documents

  1. id - ID
  2. imdb_id - ID
  3. popularity - How popular is the movie is
  4. budget - how much money spent to make the movie in dollar(\$)
  5. revenue - how much money the movie earned by dollar(\$)
  6. original title - movie title
  7. cast - which actor was casted?
  8. homepage - the movies main hompage
  9. director - who directed the movie?
  10. tagline - representative phrase
  11. keywords - some words that could be keywords of the movie
  12. overview - short summary of the movie
  13. runtime - how long the movie runs?
  14. genres - movie genre
  15. production companies - which company made the movie?
  16. release date - MM/DD/YY format. what date does the movie had released?
  17. vote count - how many people voted on this movie for assessment
  18. vote average - what is average score of movie as a result of voting?
  19. release year - YYYY format. what year does the movie has released?
  20. budget_adj - calcuated the budget as of 2010 dollar value. By inflation the dollar value goes lower.
  21. revenue_adj - calcuated the revenue as of 2010 dollar value.

The dataset is mostly consisted with text and seems the cast, genres and production_companies has combined texts that is seperated by |. The numeric data seems less. If we are going to only use numeric data it will not be hard but if we are going to analysis combined text like cast it will going to be more advanced.

# make a function to handle relation between object data type and profit
def profit_relation(df, c):
    """
    args
        df - dataframe
        c - column name
    return
        new_df - new generated dataframe
    """
    lens = df[c].str.split('|').map(len)

    # Create a new dataframe
    new_df = pd.DataFrame({'profit': np.repeat(df['profit'], lens),
                           c: list(chain.from_iterable(df[c].str.split('|')))})
    
    new_df.sort_values(by='profit', axis=0, ascending=False, inplace=True, kind='quicksort')

    # Reset the index
    new_df.reset_index(drop=True, inplace=True)
    return new_df
